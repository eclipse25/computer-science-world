## **허브, 스위치, 라우터**

<br>

네트워크 장치는 데이터를 전송하는 방식과 계층별 역할에 따라 여러 종류가 있으며,  
대표적으로 **허브(Hub)**, **스위치(Switch)**, **라우터(Router)** 가 있다.

<br>

### **허브 (Hub)**

- 허브는 네트워크에서 가장 기본적인 연결 장치이다.
- 물리 계층(1계층, Physical Layer) 또는 일부 스마트 허브는 **데이터 링크 계층(2계층, Data Link Layer)** 에서 작동한다.
- 허브에 연결된 모든 장치에게 동일한 데이터를 브로드캐스트(동시 전송)한다.
- 패킷을 특정 장치로 전송하는 기능이 없어, 트래픽이 증가하면 네트워크 성능이 저하될 수 있다.

#### **동작 방식**
- 한 장치에서 허브로 데이터를 전송하면, 허브는 이를 모든 포트로 뿌려준다.
- 데이터는 목적지가 아닌 장치에도 전달되지만, 해당 장치는 자신이 받아야 할 패킷인지 확인 후 무시한다.
- 이 방식은 **충돌(Collision)** 을 유발하여 속도를 저하시킨다.

#### **단점**
- 네트워크 트래픽이 증가할수록 충돌(Collision)이 발생하여 성능이 저하됨.
- 보안성이 낮음(데이터가 모든 장치에 노출됨).
- 대역폭을 모든 장치가 공유하여 대역폭 낭비가 발생.

#### **사용 예시**
- 과거 유선 네트워크에서 사용되었으나, 현재는 대부분 스위치로 대체됨.
- 네트워크 패킷을 단순히 확장할 때(예: 실험실, IoT 네트워크 등) 사용될 수 있음.

<br>

### **스위치 (Switch)**

- **데이터 링크 계층(2계층, Data Link Layer)** 에서 작동하는 장치이다.
- 허브보다 지능적이며, **연결된 장치의 MAC 주소를 학습하여 데이터를 필요한 목적지로만 전송**한다.
- 즉, **패킷을 목적지 장치에만 전송(유니캐스트, Unicast)** 하여 네트워크 효율성을 향상시킨다.
- 네트워크 트래픽을 관리하여 **충돌을 최소화(Collision Domain 분리)** 한다.

#### **동작 방식**
- 스위치는 **MAC 주소 테이블**을 관리하여 데이터의 목적지를 학습한다.
- 한 장치에서 데이터가 전송되면, **목적지 MAC 주소를 조회하여 해당 장치가 연결된 포트로만 데이터 전송**한다.
- 처음 연결된 장치는 MAC 주소를 알 수 없으므로, 스위치는 **브로드캐스트(Broadcast) 방식**으로 전송 후 학습한다.

#### **장점**
- **네트워크 효율성 증가**: 불필요한 데이터 전송을 줄여 대역폭 낭비를 방지한다.
- **충돌 방지**: 각 포트가 독립적인 충돌 도메인을 가지므로, 허브보다 충돌 가능성이 낮다.
- **보안 강화**: 데이터가 특정 장치에만 전달되어 허브보다 보안성이 뛰어나다.

#### **단점**
- MAC 주소를 저장하고 학습하는 기능으로 인해 **허브보다 가격이 비싸다**.
- 네트워크 트래픽이 많을 경우 **처리 부하가 발생할 수 있다**.

#### **사용 예시**
- 대부분의 **LAN(Local Area Network, 근거리 통신망)** 에서 기본 장비로 사용됨.
- 기업, 데이터센터 등에서 **고속 네트워크 구축**을 위해 사용됨.

<br>

### **라우터 (Router)**

- **네트워크 계층(3계층, Network Layer)** 에서 작동하는 장치이다.
- **IP 주소 기반**으로 데이터를 라우팅하여, **서로 다른 네트워크를 연결**하는 역할을 한다.
- WAN(인터넷)과 LAN(사내 네트워크)를 연결하는 역할을 수행한다.

#### **동작 방식**
- 네트워크 간 **IP 주소를 기반으로 최적의 경로를 찾아 패킷을 전달**한다.
- 라우터는 **라우팅 테이블(Routing Table)** 을 참고하여 목적지 네트워크로 패킷을 전송한다.
- **NAT(Network Address Translation)** 기능을 활용하여 **사설 IP를 공인 IP로 변환**하여 인터넷 접속을 가능하게 한다.

#### **장점**
- **서로 다른 네트워크 연결 가능**: 네트워크 간 데이터 전송 및 인터넷 연결을 관리한다.
- **보안 기능 제공**: NAT 및 방화벽(Firewall) 기능으로 내부 네트워크를 보호한다.
- **트래픽 제어**: 트래픽을 관리하여 최적의 경로로 패킷을 전달한다.

#### **단점**
- **스위치보다 속도가 느리다**(라우팅 경로를 계산해야 하기 때문).
- 네트워크 설정이 상대적으로 복잡하다.
- 가격이 상대적으로 높다.

#### **사용 예시**
- 가정용 및 기업 네트워크에서 **인터넷 연결을 제공**하는 장치.
- 사설 네트워크와 인터넷을 연결하는 **공유기(Wireless Router)** 에 포함됨.
- 데이터센터, ISP(인터넷 서비스 제공업체)에서 **대규모 네트워크 관리**에 사용됨.

<br>

### **차이점 비교**

| 구분 | 허브 (Hub) | 스위치 (Switch) | 라우터 (Router) |
|------|------------|----------------|----------------|
| **동작 계층** | 물리 계층 (L1) | 데이터 링크 계층 (L2) | 네트워크 계층 (L3) |
| **데이터 전송 방식** | 브로드캐스트 | 유니캐스트 (MAC 주소 기반) | 유니캐스트 (IP 주소 기반) |
| **트래픽 관리** | 없음 (모든 장치로 전송) | 목적지 장치로만 전송 | 최적의 경로로 라우팅 |
| **충돌 도메인** | 하나 (모든 장치 공유) | 각 포트별 독립적 | 각 네트워크별 독립적 |
| **브로드캐스트 도메인** | 하나 (전체 네트워크) | 하나 (VLAN 적용 시 분리 가능) | 네트워크별 분리 |
| **보안성** | 낮음 (패킷이 모든 장치에 전송됨) | 중간 (패킷이 특정 목적지에만 전송됨) | 높음 (방화벽, NAT 지원) |
| **속도** | 낮음 (충돌 많음) | 중간 (MAC 주소 기반 전송) | 상대적으로 낮음 (라우팅 계산) |
| **가격** | 저렴함 | 중간 | 비쌈 |
| **사용 예시** | 실험용 네트워크 | 기업, 가정 LAN 네트워크 | 가정 및 기업 인터넷 연결 |

<br>

### **결론**
- 허브는 단순한 장치로, 현재는 거의 사용되지 않는다.  
- 스위치는 LAN 환경에서 필수적인 네트워크 장비이다.  
- 라우터는 네트워크 간 연결 및 인터넷 접속을 담당한다.  
- 일반적으로 **가정 및 사무실에서는 라우터에 스위치 기능이 포함된 공유기**를 사용한다.
